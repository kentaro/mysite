'02/09/19_a (thu)

フラゲ って言葉はわりと一般的だったりするのですか？ いや、昨日のことですが、「今日は「 The 美学」の フラゲ 日だ」みたいな記述をあちこちで読んだので。多分「フライング・ゲット」の略なのでは？ と推察したのですが。

ところで、valid な HTML によって作成された文書でもってあややを讃えることを目指している当サイトとしては、不思議マクアプ文書によるあややへの言及をすなわち冒涜と見なさざるを得ないのであります。そこで思いついたのが「あやや validator （仮」。一見したところあややを賛美しているかのようなサイトも、この validator にかければたちどころに化けの皮が剥がれてしまうという恐ろしいツール。

しかし、問題点が…。

  • HTML を判定するだけなら AHL か W3C HTML Validation Service があれば十分。
  • そこで、「あやや度」という概念を導入し、どれほどあややに対する思いが深いかを判定する必要がある。
  • つか、あやや度ってなに？ その基準は？
  • というよりもむしろ、このサイトが invalid と判定されてしまう可能性大。それは困る。

今日はずっと「あやや度とはなにか？」ということばかり考えてたので、お仕事がぜんぜん手につきませんでした。頭の中では「 The 美学」がエンドレスで流れているし。もう聴きまくりすぎて、頭の中でなってる音楽と CD からの音楽との区別が付きません。ある意味、便利ですけど。

モーヲタ住基ネット に登録してみるテスト。つか、バナーまわりのマークアップが適当ですが、まぁいいや。

とりあえず、あやや validator の原型を作ってみました。

つか、W3C HTML Validation Service の判定結果ページのソースの各行を配列に格納して、必要なところだけを echo するようにしただけ、というかなりいいかげんな作りです…。ていうかそれって、見つかったらヤバいですよ＞けんたろたん。思いっきり著作権を無視してます。しかもその行数を正規表現で判定しているわけではなく、決め打ちで指定しているので、なんと！ validator なのに、valid でないサイトを判定すると表示が著しくおかしくなってしまうという罠。つまり、あらかじめ valid であることがわかっているサイトを判定させなければならないという、腐った仕様。まぁ、実際どんな風になるか、お試しください。

いやもう、全く以て未完成なのですが、疲れたので今日はここまで。つか、僕はなにをやってるんだろう…？ という気持ちになりますた。

と思ったけど、またいじってみました。改造の結果…判定にかけたすべてのサイトが valid になってしまうというものすごい仕様になってしまいました！ まさしく、「全宇宙あやや化計画」を推進するにふさわしいツールといえましょう…(>_<)
